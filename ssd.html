Secure Software Development

Unit 1: Introduction to Secure Software Development


Learning Objectives
  1. Identify relevant academic literature which discusses the ways in which secure software may be developed using an agile process.
  2. Decompose an activity into a set of tasks which can be represented using a flow chart from the UML portfolio of modelling techniques.
  3. Describe the importance of creating a culture of risk awareness in an organization, in addition to the creation of secure software.

This module focused on the Security aspect of Software development. In this Unit, the concept of Secured design was introduced. This started by discussing the various approaches to Software design as well as the Architecture of software design. The modelling processes are also discussed and a Learning discussion forum was introduced on UML flowchart.

The OWASP(Open Web Application Security Project) is a collection of independent software and security experts with a mandate to improve the software Security. Renowned information technology companies like Panasonic, Accenture and Allstate are participating members of this Project. OWAPS Identified 10 top software security weaknesses as Broken Access Control, Cryptographic Failures, Injection, Insecure Design, Security Misconfiguration, Vulnerable and Outdated Components, Identification and Authentication Failures, Software and Data Integrity Failures, Security Logging and Monitoring Failures and Server-Side Request Forgery (Owasp.org, 2021). also Identified by OWASP is the Top 10 Proactive controls; Define security requirements, Leverage security frameworks and libraries, Secure database access, Encode and escape data, Validate all inputs, Implement digital identity, Enforce access controls, Protect data everywhere, Implement security logging and Implement digital identity, Enforce access controls, Protect data everywhere, Implement security logging and monitoring and Handle all errors and exceptions(Owasp.org, n.d.).

The software development Life Cycle has two approaches; The predictive and the Adaptive Approach. Waterfall Model is a predictive approach while the Agile model is an Adaptive approach. Security issues with the Waterfall model is its in ability to accommodate immediate changes, user and client inputs and delay in Testing. Security issue with the Agile Model is the pressure it suffers from demand for a working programme at each stage of the process which leads to inadequate testing. Information security knowledge may also be missing throughout the process as emphasis is on quality and suitability

The Architecture of a system is the fundamental Organisation of its components and the relationship with each other as well as its environment, stakeholders and the principle guiding its design and evolution(web.archive.org, 2011). The TOGAF was designed by the Open group and has been used by most organizations as their Architecture framework. This framework is a 10 phase process.

The TOGAF is now modified with SABSA following a critism that it fails to address security in its framework.

Design patterns are used to describe in an organised format, how the code will proffer solutions to the frequently identified software engineering problems. This is structured as a Creational Factory,Structural Adapter, Behavioral Mediator. Antipatterns come in when the pattern considered appropriate solution to a problem does not exist

The Unified Modelling Language(UML) provides information or describes the process towards the development of the solution in a graphical language. UML consists of classifiers, Event and Behaviors. The classifiers define the set of objects which usually have a common feature. The Event describes the occurrences within the identified system and the behaviours describe the kinetics and reactions to these Events. Examples of UML diagrams are Use cases, Sequence Diagrams, Class Diagrams and Activity Diagrams. UMLs enable communication during the software development life cycle

Unit 3: Introduction to Programming Languages


Learning outcomes
  1. Describe some key milestones in the development of programming languages.
  2. Outline some of the key paradigms that define the different types of languages.
  3. Explain the key concepts that determine the operation of programming languages.
  4. Discuss key programming challenges and recommended best practices.
  5. Explain what design patterns are and when to use them.

Programming Languages are languages prepared by humans to enable communication with computer systems. A programming language is made abstract to make it easier for humans to create programs that affect outcomes on computing devices. Although programming languages are readable by humans, they are not directly executed in the computer but are transformed into machine native code. There are 2 approaches to this conversion; either via compilers or Interpreter.

A compiler takes as input that is the source program written in the programming language and converts it to a machine readable program called the Target program all at once. The Interpreter does this one statement at a time. Modern programming languages like Java and C# are however a hybrid of these systems.

Agile comes with a manisfesto which focus on individuals and interactions instead of process and tools, Working product instead of Comprehensive documentation, customer collaboration instead of Contract Negotiations, Responding to change instaed of following a plan. Models include DSDM (dynamic systems development method), RUP (the rational unified process)

Programming Paradigms are classified as imperative( programs are structured sequentially, Functional(based on evaluation and expression of math formulas), Logical(Artificial intelligence), Object Oriented(Uses abstractions).

There are several concepts used to describe programming languages. These concepts are based on the following terminologies; Abstraction, Casting, Encapsulation, Garbage collection, Grammar, Inheritance, Mutability, Pointers, Polymorphism, Queues, Recursion, Regex, Stacks and Syntax. Python is one programming language that can be used to describe these concepts. Python is designed to be an easily readable and has found more users because of it is easy to understand by beginners.

Software testing is important to avoid failures. Validation and verification is used in software testing. Whitebox and blackbox approach can be used in software testing. In whitebox approach, the tester knows the instructions of the code while in blackbox approach, it is not known to the tester.Whitebox is basically used for internal system testing eg unit testing while blackbox for overall functionality of the system eg equivalence testing. Software testing should be carried out early and throughout the lifecycle. the testing process defined by the ISO/IEC/IEEE in 29119 Part 4 can be summarised as; identify feature sets, Derive test conditions, Derive test coverage Items, Derive test cases, Assemble test sets, Derive test procedures.

Testing types include, functional Testing, Usability testing, compatibility testing, Regression testing, Acceptance Testing, Performance testing, Big bang testing. The major broad testing types are Vulnerability testing and Penetration testing.

The Cyclomatic Complexity can be used to determine how complex our code is. It involves counting the number of decision points in code. Code complexity increases with the number of decision points. The complexity inturn indicates the more likelihood of errors or problems in the code. Myers' Interval shows how boolean operators are used in much the same way that the more operands used, the more its complexity and also its chance of errors.

Unit 5: Future trends in Secure Software Development


Learning outcomes
  1. Give examples of fog computing, IoT and Cyber Physical System components and solutions.
  2. Describe some of the security issues with the above-mentioned systems.
  3. Recommend emerging technologies and solutions to investigate.

This unit discussed the future trends relevant to software technologies, focusing on security in emerging technologies like blockchain, cyber physical systems, microservice systems, programming languages like Rust, Swift and Go(lang), as well as the legal, ethical, social and professional considerations concerned with the use and development of software systems. The GDPR and other laws regarding software designs are also changing the landscape The security standpoint in software technologies gives more consideration to processes as secured Software development life cycles and DevOps. More focus on software testing, risk analysis, as well the automation of security processes will reduce the impact of cybercrime.
